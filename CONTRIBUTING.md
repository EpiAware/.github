# Contributing to EpiAware

[![ColPrac: Contributor's Guide on Collaborative Practices for Community Packages](https://img.shields.io/badge/ColPrac-Contributor%27s%20Guide-blueviolet)](https://github.com/SciML/ColPrac)

Thanks for contributing to an EpiAware package.
These guidelines apply across the [EpiAware](https://github.com/EpiAware)
organisation.

## Collaborative practices

We follow the
[ColPrac: Contributor's Guide on Collaborative Practices for Community
Packages](https://github.com/SciML/ColPrac).
Read it before opening an issue or pull request.
Where a package documents its own conventions, those take precedence over
this guide; otherwise ColPrac applies.

Packages are encouraged to display the ColPrac badge in their README:

```markdown
[![ColPrac: Contributor's Guide on Collaborative Practices for Community Packages](https://img.shields.io/badge/ColPrac-Contributor%27s%20Guide-blueviolet)](https://github.com/SciML/ColPrac)
```

## How to contribute

- Open an issue before starting non-trivial work so we can agree the
  approach.
- Fork the repository and work on a branch.
- Keep pull requests focused; one logical change per pull request.
- Add or update tests for any behaviour you change.
- Run the package's tests and formatter locally before pushing.
- Reference the issue your pull request closes.

## Code of Conduct

By participating you agree to abide by our
[Code of Conduct](CODE_OF_CONDUCT.md).

## Reporting security issues

Do not report security vulnerabilities through public issues.
See [SECURITY.md](SECURITY.md) for how to report them privately.
